1. PROJECT OVERVIEW
================================================================================

H2S is a Smart Crisis Management System built as a full-stack web application.

Tech Stack:
  - Backend:  Node.js, Express (ESM), Morgan, Dotenv
  - Frontend: React 18, Vite, React Router 6, CSS Modules
  - Standard: ES6+ (import/export) throughout — no CommonJS

================================================================================
2. ARCHITECTURE OVERVIEW
================================================================================

The system follows a strict Layered Clean Architecture pattern:

  Request → Route → Controller → Service → Repository → Database
  Response ← Route ← Controller ← Service ← Repository ← Database

Key Principles:
  - Each layer has a single, well-defined responsibility.
  - Dependencies flow inward (routes depend on controllers, controllers on
    services, services on repositories). Never the reverse.
  - No layer may skip another layer (e.g., controllers must NOT query the
    database directly).

3. BACKEND STRUCTURE
================================================================================

backend/
  ├── config/              Centralized configuration (ports, env variables)
  │   └── config.js        Reads from .env / provides defaults
  │
  ├── routes/              Express Router definitions
  │                        Maps HTTP endpoints to controller functions.
  │                        No business logic here — just routing.
  │
  ├── controllers/         Orchestration layer
  │                        Receives requests from routes, coordinates between
  │                        services, and returns responses.
  │                        Think of it as the "conductor" — it doesn't do the
  │                        work, it tells others what to do and assembles results.
  │
  ├── services/            Pure business logic
  │                        Like DSA problems: receives input, produces output.
  │                        MUST NOT query the database.
  │                        MUST NOT call other services.
  │                        Focuses on computation/transformation only.
  │
  ├── repository/          Data Access Layer
  │                        Each repository "owns" one table/model.
  │                        All database queries for that table live here.
  │                        Custom query functions (e.g., getUserById,
  │                        getUsersByRole) are defined here.
  │                        If another model needs data from this table, it must
  │                        go through this repository.
  │
  ├── middlewares/         Express middleware
  │   ├── errorMiddleware.js   Centralized error handler
  │   └── loggerMiddleware.js  Request timing logger
  │
  ├── utils/               Shared utilities
  │   ├── response.js          Standard success/error response helpers
  │   └── responseWrapper.js   Alternative response wrapper (ESM)
  │
  │  ├── initialiser_helpers/  Maintenance and seed scripts
  │   ├── seed_admin.js         Initial admin account creator
  │   └── test_auth.js          CLI authentication tester
  │
  ├── app.js               Express app setup (middleware, routes, error handler)
  └── server.js            Entry point — starts the HTTP server

5. DATA FLOW
================================================================================

BACKEND (per request):

  1. Route         → Receives HTTP request, delegates to controller
  2. Controller    → Orchestrates: calls repository for data, passes to service
                     for processing, sends response
  3. Service       → Receives data, performs logic/transformation, returns result
  4. Repository    → Executes specific database query, returns raw data

FRONTEND (per user action):

  1. Page/Component → Calls apiService function
  2. apiService     → Calls apiClient with endpoint
  3. apiClient      → Performs fetch(), handles errors, returns parsed JSON
  4. useFetch hook  → Manages loading/error/data state + refetch capability

6. DEVELOPMENT CONVENTIONS
================================================================================

  - ESM Only:         Use import/export everywhere. No require/module.exports.
  - File Extensions:  Always include .js in import paths (backend Node ESM).
  - CSS Modules:      All component styling uses .module.css for isolation.
  - No Inline Styles: Zero inline CSS in React components.
  - One Repo = One Table: Each repository file owns exactly one DB table.
  - Services are Pure: No DB calls, no side effects, no calling other services.
  - Controllers Orchestrate: They glue services and repositories together.
